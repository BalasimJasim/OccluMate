

Here’s a high-level instruction guide for Cursor AI to improve the patient portal dashboard by making it functional, user-friendly, and visually cohesive. We’ll specify the relevant files and components to help Cursor AI focus on necessary modifications.

High-Level Instructions for Improving the Patient Portal Dashboard
Objective:
Enhance the patient portal experience by making the dashboard intuitive, functional, and visually appealing. Ensure each feature is accessible and user-friendly, particularly for booking appointments, viewing prescriptions, accessing documents, and adjusting settings.

1. Make Dashboard Buttons Functional and Intuitive
File Focus: frontend/src/components/PatientPortal/PortalDashboard.jsx and frontend/src/pages/PatientPortal.jsx
Tasks:
Ensure each button (Appointments, Prescriptions, Documents, Settings) links to the corresponding sections.
Add hover effects and visual cues to make buttons appear interactive.
Provide brief, descriptive tooltips on each button to improve user understanding.
2. Improve Appointment Management
File Focus: frontend/src/components/Appointments/AppointmentBooking.jsx, AppointmentDetails.jsx, AppointmentScheduling.jsx
Tasks:
Enable functionality for booking new appointments and canceling existing ones.
Display an appointment calendar, where patients can view upcoming appointments.
Integrate a "confirm cancellation" dialog before deleting an appointment.
3. Optimize Prescriptions Access
File Focus: frontend/src/components/Prescriptions/PrescriptionForm.jsx, PrescriptionManagement.jsx
Tasks:
Display a list of prescriptions for the patient with options to view details.
Include a button that enables the patient to request prescription refills.
Add sorting options (e.g., by date) for better organization.
4. Enable Document Management
File Focus: frontend/src/components/PatientPortal/DocumentManager.jsx
Tasks:
Allow patients to upload and download documents related to their medical history.
Display document titles, upload dates, and file types for easy identification.
Provide functionality for patients to delete documents, with confirmation prompts.
5. Refine Settings in the Patient Portal
File Focus: frontend/src/components/PatientPortal/PortalSettings.jsx
Tasks:
Allow patients to update personal information and notification preferences.
Add functionality to change the password.
Integrate validation for form fields to prevent incorrect data entry.
General UX/UI Enhancements
Styles: Improve the overall aesthetic using frontend/src/styles/components/_patient.scss.
Buttons & Forms: Implement consistent button styles and input forms, using files in frontend/src/styles/abstract for mixins and variables.
Navigation & Layout: Optimize the layout for a clear, modern look using frontend/src/components/Navigation/Navbar.jsx and Sidebar.jsx.
