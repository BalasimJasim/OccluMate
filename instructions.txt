##Instructions for Cursor AI: Building the OccluMate Project:

###Project Overview
Project Name: OccluMate
Description: OccluMate is a full-featured web app designed to manage dental office operations efficiently. It includes features for patient data management, appointment scheduling, dental chart management, prescription handling, and task management. The goal is to create an intuitive, organized, and scalable system for use by dental clinics.

###Features Requirements
1-User Authentication

Implement secure user registration, login, and logout functionalities with JWT for authentication.
Role-based access: differentiate user types (Admin, Dentist, and Receptionist) with access controls.
2-Patient Management

Enable CRUD operations for patient records, including personal details, contact information, and 3-medical history.
Build a user-friendly interface for updating patient profiles.
3-Appointment Scheduling

Create a calendar-based system for managing patient appointments.
Implement reminders, rescheduling, and cancellation options.
4-Dental Chart Management

Provide a graphical interface for updating and viewing patient dental charts.
Store detailed records of dental procedures and diagnoses.
5-Prescription Management

Enable dentists to create, update, and view prescriptions.
Integrate prescription history into patient records for easy access.
6-Task Management

Design a task management feature for tracking daily tasks (e.g., follow-ups, calls).
Allow task assignment to specific team members with due dates and priority labels.
7-Analytics

Add a dashboard to display analytics on patients, appointments, treatments, and task completion rates.

###Relevant Documentation:
MERN Stack Documentation

MongoDB for database design and queries.
Express.js for backend routing and middleware setup.
React.js for creating interactive and responsive frontend components.
Node.js for server-side logic.
Project Libraries

Authentication and security: jsonwebtoken, bcryptjs.
API calls: axios.
State management: React Context API or Redux if needed.
UI components: any specific libraries for UI elements like charts or form handling.
Backend Documentation

authMiddleware.js: For enforcing JWT-based authentication across all routes.
Controllers and models for each module (patients, appointments, dental charts, prescriptions, tasks).
Best practices for error handling and validation in Express.

Authentication System

Patient Management

Appointment System

Style the appointment scheduler interface and integrate it within Dashboard.jsx.
Dental Chart and Prescription

Develop components under frontend/src/components/Auth/ for dental chart and prescription functionalities.
Build dentalChartController.js and prescriptionController.js to handle the respective backend logic.
Integrate views within Dashboard.jsx and Home.jsx.
Task Management

Complete taskController.js and TaskManagement.jsx to track and manage tasks.
Create task components in frontend/src/components/Auth/TaskManagement.jsx.
Style within _dashboard.scss and _forms.scss as needed.
Analytics Dashboard

Use Analytics.jsx to display patient statistics, treatment data, and task progress.
Design with UI charts and graphs for a responsive analytics page.
